# V1 Requirements & Status

## Original Requirements

Tracing/investigations tool for blockchain/cryptocurrency.
- Underlying data pulled — who do we get labelled data from?
- What features/functions do we have?
- Graph generation, demonstrative creation. Labelling of transactions etc.
- Do we allow graph ingestion from other 3P sources?
- References Cited aggregation/document response tool
- Expert gets subpoena for deposition, gets a records request, the tool has an export feature to list out all transactions referenced, sources used, etc.
- Report writing
- Drafting report using the primary source materials/tracing information produced by the platform.
- Generate the report, works cited, charts in legible format, exhibits etc.
- Shared workspace — intelligence platform for crypto cases/law firms
- Provide a shared workspace for lawyers and experts to collaborate, ask questions, inquire about findings.

---

## Status (as of 2026-04-25)

| Requirement | Status | What's Built |
|---|---|---|
| Tracing/investigations tool | Done | Graph canvas (Cytoscape), traces, nodes, edges, staging panel, batch edit |
| Labeled data | Done | Global LabeledEntities registry, Daubert Label badges on nodes, admin CRUD at `/admin/entities` |
| Graph generation | Done | Agent creates/mutates graphs via `execute_script` + `graph-mutations` skill |
| Demonstrative creation (charts, reports, chronologies) | Done | Productions system — reports (TipTap HTML editor), charts (Chart.js), chronologies (table with source links) |
| Report writing | Done | Agent creates HTML reports via `create_production` tool, TipTap WYSIWYG with view/edit toggle |
| Works cited / references | Done | Citation system — `[*]` inline markers, cheerio extraction at export, auto-generated Works Cited appendix |
| Export (transactions referenced, sources used) | Done | PDF/HTML export for all production types + graph snapshots via server-side Puppeteer |
| Admin panel | Done | `/admin` — users, cases, entities management with `IsAdminGuard` |
| Shared workspace | Done (V1) | CaseMember roles (owner/guest), case-level access control, shared investigations/productions/chat. Real-time collaboration deferred to V2. |
| Data Room (document storage integration) | Not built | Plan at `docs/plans/DataRoom.md`. OAuth to Google Drive/Dropbox not implemented. |
| 3P graph ingestion | Deferred (V2+) | No CSV/JSON import UI. Only agent-driven via Etherscan/Tronscan APIs. |
